New World

I've been a tester on New World for over a year. Working on the engine QA team, I learned about MMOs, and the engine's underlying systems as developers make changes to it. It was also my first experience using Python to write a functional tool for other people to use for bot automation.

Much of what the engine team handles covers changes to the game's core systems. I've learned about how rendering systems work on a technical level, many industry standard editor features used for modern 3D AAA games, server/multiplayer systems, running and writing bot scripts for automation, and the daily life of an engineer.

I already had experience writing code from college, but this job allowed me to actually apply it to create something. This job also expanded my experience to using the Jira database, writing documentation through Confluence, and the repository system Perforce. I learned how to deploy data planes to servers and builds to Steam, and used Jenkins for automation.

The biggest challenge of this project has to have been drinking from the firehose of information that was required before I could start doing impactful work. There are several interdependent systems and processes that don't always make sense in a vacuum, so much of my onboarding involved taking lots of notes for myself. Once I understood our processes on the whole, I actually preferred tasks that challenged me to understand technical concepts, and specialized in testing systems that other testers were daunted by.

New World is the second game I've been a long-term QA tester for. It's the most hands-on I've ever been with a AAA game up to this point, and my first experience coding a graphical program to be used by a team. I'm in dev meetings every morning, and in direct contact with engineers almost every day; I've been given a window into what will be expected of me when I take on an engineer role. I've met a lot of great people, made solid connections, and expanded my repertoire of skills and knowledge.
